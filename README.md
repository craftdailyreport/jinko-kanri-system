## STEP 2：Google Apps Scriptを設定する

1. スプレッドシートのメニュー「拡張機能」→「Apps Script」を開く
2. デフォルトで開かれているコードをすべて削除
3. `gas-code.gs.txt` の内容を全てコピーして貼り付け、保存（Ctrl+S）
4. 右上の「デプロイ」→「新しいデプロイ」をクリック
5. 「種類の選択」（歯車アイコン）→「ウェブアプリ」を選択
6. 以下を設定：
   - 説明：人工管理システム（任意）
   - 次のユーザーとして実行：**自分**
   - アクセスできるユーザー：**全員**
7. 「デプロイ」をクリック → 初回は権限の承認が必要（「許可を確認」→ Googleアカウントを選択→「詳細」→「（プロジェクト名）に移動」→「許可」）
8. 表示された **ウェブアプリのURL** をコピーする（`https://script.google.com/macros/s/.../exec` の形式）

## STEP 3：HTMLファイルにURLを設定する

`index.html` を開き、以下の行を探して書き換えます。

```javascript
const GAS_URL = 'ここにGASのWebアプリURLを貼り付け';
```

↓ STEP2でコピーしたURLに置き換える

```javascript
const GAS_URL = 'https://script.google.com/macros/s/AKfycby.../exec';
```

## STEP 4：Webアプリを公開する

`index.html` 1ファイルだけで動作します。以下のいずれかの方法で公開してください。

### 方法A：GitHub Pages（無料・推奨）
1. GitHubでリポジトリを新規作成
2. `index.html` をアップロード
3. リポジトリの Settings → Pages → Source で「main」ブランチを選択して保存
4. 数分後、`https://ユーザー名.github.io/リポジトリ名/` で公開されます

### 方法B：レンタルサーバー
- 契約しているレンタルサーバーのFTP/管理画面で `index.html` をアップロードするだけで公開できます

### 方法C：社内サーバー・NAS
- 社内のファイルサーバーで静的ファイル配信ができる場合はそこに置くだけでも利用可能です

## 動作確認

1. 公開したURLを開く
2. マスタ_ユーザーシートに登録したID・パスワードでログイン
3. 作業員でログイン → 日報を提出 → スプレッドシートの「日報」シートに行が追加されることを確認
4. 管理者でログイン → 「日報一覧」に反映されることを確認

## 注意点・補足

- **GASのno-cors送信について**：このアプリはGASへの送信に `mode: 'no-cors'` を使用しています。これはブラウザのCORS制限を回避するための標準的な方法で、送信は正常に行われますが、レスポンス内容は読み取れません（送信が成功したかどうかはネットワークタブで確認できます）。
- **パスワードの平文管理について**：現状はスプレッドシートにパスワードを平文で保存する簡易仕様です。セキュリティを強化したい場合は、ハッシュ化（GASの`Utilities.computeDigest`等）の実装をご検討ください。社内利用など限定環境であれば現状でも問題ない場合が多いです。
- **マスタの編集**：物件略名・作業項目・作業者・時間の追加/編集/削除は、現状はスプレッドシートを直接編集する運用です。アプリ上での編集UIが必要な場合は追加実装が可能です。
- **同時アクセス**：GASは大量同時アクセスに弱い面があるため、数十人規模までの利用を想定しています。
